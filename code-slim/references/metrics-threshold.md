# 量化阈值

> 什么算"胖"的标准。扫描时以此为判据，避免主观判断。

## 文件级阈值

| 指标 | 阈值 | 超过则 |
|------|------|--------|
| 单文件行数 | > 500 行 | 标记为 God File 候选 |
| 单文件行数 | > 300 行 | 标记为 Large File，需关注 |
| 文件职责数 | ≥ 3 个独立职责 | 标记为需要拆分 |
| 同目录文件数 | > 15 个 | 考虑子目录分组 |

### 职责判定方法
一个"职责"= 一组操作同一数据/做同一类事的函数。如：
- DB CRUD 是一个职责
- API 响应构建是一个职责
- 业务计算是一个职责
- 日志/监控是一个职责

## 函数/组件级阈值

| 指标 | 阈值 | 超过则 |
|------|------|--------|
| 函数行数 | > 80 行 | 标记为 God Function |
| 函数行数 | > 40 行 | 标记为 Long Function，需关注 |
| 函数参数 | > 6 个 | 考虑用 options 对象 |
| 嵌套深度 | > 4 层 | 需要提取子函数 |
|做的事 | ≥ 3 件 | 标记为需要拆分 |

### React 组件特殊阈值

| 指标 | 阈值 |
|------|------|
| 组件行数 | > 300 行需拆分 |
| useState 数量 | > 5 个考虑 useReducer |
| useEffect 数量 | > 3 个考虑自定义 Hook |
| Props 数量 | > 8 个考虑组合模式 |
| 条件渲染分支 | > 5 个考虑状态机 |

## 重复代码阈值

| 指标 | 阈值 |
|------|------|
| 相似代码出现次数 | ≥ 3 次 |
| 相似代码行数 | ≥ 5 行 |
| 相似度 | > 80%（参数名可不同，结构相同即可） |
| 合并后预估削减 | > 10 行（太小不值得提取） |

## Token 浪费估算

### 估算公式
```
浪费 token ≈ 浪费行数 × 10（Python）/ 15（TypeScript）
```

### 分级
| 浪费等级 | 估算 token | 建议动作 |
|---------|-----------|---------|
| 低 | < 500 | 可选优化 |
| 中 | 500 - 2000 | 建议优化 |
| 高 | 2000 - 5000 | 推荐优化 |
| 严重 | > 5000 | 必须优化 |

### 浪费来源权重
| 来源 | 每行权重 |
|------|---------|
| 死代码（整个文件） | × 1.0 |
| 死代码（函数/方法） | × 0.8 |
| 重复代码（每多一份） | × 0.7 |
| 过度设计（可简化） | × 0.5 |
| 注释掉的代码 | × 1.0 |

## 项目整体健康度

| 指标 | 健康 | 警告 | 危险 |
|------|------|------|------|
| 死代码占比 | < 2% | 2-5% | > 5% |
| 重复代码占比 | < 3% | 3-8% | > 8% |
| God 文件占比 | < 5% | 5-15% | > 15% |
| 平均文件行数 | < 150 | 150-300 | > 300 |

### 占比计算
```
死代码占比 = 死代码行数 / 项目总代码行数
项目总代码行数 = find . -name "*.py" -o -name "*.ts" -o -name "*.tsx" | xargs wc -l
```

# SEO 排名提升研究报告 (2024-2026)

**适用对象**: 中国 SaaS/内容平台（百度 + Google 双引擎）
**数据来源**: Google Search Central 官方文档、web.dev、百度搜索资源平台、行业实证研究

---

## 1. 内容质量信号

### Google "高质量"判定标准

| 因素 | 建议 | 优先级 |
|------|------|--------|
| 原创性 | 必须提供原创信息、报道、研究或分析，非简单复制改写 | **Must-do** |
| 完整性 | 对主题提供"实质性、完整或全面"描述 | **Must-do** |
| 深度洞察 | 超越显而易见的内容，提供有洞察力的分析 | **Must-do** |
| 附加价值 | 引用其他来源时必须提供"实质性的额外价值和原创性" | **Must-do** |
| 无最低字数 | Google 明确声明"没有魔法字数目标"，但程序化页面建议 ≥300 字 | 注意 |
| E-E-A-T | Experience/Expertise/Authoritativeness/Trustworthiness，Trust 最重要 | **Must-do** |

### 百度 "高质量"判定标准

| 因素 | 建议 | 优先级 |
|------|------|--------|
| 内容原创度 | 百度对采集惩罚比 Google 更严格；飓风算法专门打击采集站 | **Must-do** |
| 时效性 | 百度对内容时效性极其敏感，权重高于 Google | **Must-do** |
| 页面可读性 | 落地页体验检测，对广告干扰容忍度极低 | **Must-do** |
| 实质性内容占比 | 主体内容占页面总面积的比例影响质量评分 | **Must-do** |

### 聚合/策展内容

| 建议 | 说明 | 优先级 |
|------|------|--------|
| 必须有增量价值 | 单纯聚合 = "scaled content abuse"（规模化内容滥用） | **Must-do** |
| 添加编辑观点 | 加入专业评论、对比分析、专家观点 | **Must-do** |
| 标注来源 | 明确标注原始来源，加来源声明 | 推荐 |
| AI 内容披露 | Google 建议向读者披露 AI 辅助 | 推荐 |

## 2. On-Page SEO

### Title 标签

| 建议 | 详情 | 优先级 |
|------|------|--------|
| 每页唯一 title | Google 严重依赖 `<title>` 生成搜索结果标题 | **Must-do** |
| 建议格式 | `主题描述 - 品牌名` | 推荐 |
| 中文长度 | 20-35 个汉字（百度显示 ~30 字，Google ~35 字） | 推荐 |
| 避免堆砌 | Google 明确将 keyword stuffing 列为垃圾政策 | **Must-do** |

### H1/H2 结构

| 建议 | 优先级 |
|------|--------|
| 每页一个 H1，与 title 互补但不完全相同 | 推荐 |
| H1 > H2 > H3 逻辑递进 | 推荐 |
| Google: "标题使用顺序不影响排名" | 注意 |

### 关键词密度

| 建议 | 优先级 |
|------|--------|
| 自然写作，Google 用 BERT 理解语义 | **Must-do** |
| 覆盖同义词（"公众号" vs "微信公众号"） | 推荐 |
| 中文 1-2% 足够，>3% 有 stuffing 风险 | 推荐 |

### E-E-A-T 实现

| 信号 | 实现方式 | 优先级 |
|------|---------|--------|
| 作者署名 | byline + 链接到作者介绍页 | **Must-do** |
| 作者资质 | 作者页展示专业背景 | **Must-do** |
| 关于页面 | 清晰的 About 页面说明背景 | **Must-do** |
| 引用来源 | 内容中引用可信来源 | 推荐 |
| AI 披露 | 披露内容如何产生 | 推荐 |

## 3. 技术 SEO

### Core Web Vitals（2024-2026）

| 指标 | 好 | 需改进 | 差 |
|------|------|--------|-----|
| **LCP**（最大内容绘制） | ≤ 2.5s | 2.5-4s | > 4s |
| **INP**（下次绘制交互，已替代 FID） | ≤ 200ms | 200-500ms | > 500ms |
| **CLS**（累积布局偏移） | ≤ 0.1 | 0.1-0.25 | > 0.25 |

### 移动优先

| 建议 | 优先级 |
|------|--------|
| 移动端体验决定排名（Google 2019 年全面移动优先） | **Must-do** |
| 移动版必须包含与桌面版相同的内容 | **Must-do** |
| 百度搜索资源平台提交移动适配规则 | **Must-do** |

### 结构化数据

| Schema 类型 | 适用场景 | 优先级 |
|-------------|---------|--------|
| `Article` | 文章/内容页 → 富摘要 | **Must-do** |
| `Breadcrumb` | 内容层级 → 面包屑 | 推荐 |
| `FAQPage` | FAQ 页面 → FAQ 富摘要 | 推荐 |
| `Organization` | 首页 → 品牌知识面板 | 推荐 |

### 技术要求

| 项目 | 优先级 |
|------|--------|
| HTTPS（百度+Google 都要求） | **Must-do** |
| robots.txt 正确配置 | **Must-do** |
| XML Sitemap 提交到两个站长平台 | **Must-do** |
| 每页 canonical URL | **Must-do** |
| 关键内容 SSR（百度 JS 解析能力弱） | **Must-do** |

## 4. 内容新鲜度

### Google Freshness Systems

Google 有专门的新鲜度系统，对以下查询给予更新内容更高排名：
- 最近发生的事件
- 定期重复的事件
- 经常更新的信息

### 更新策略

| 建议 | 优先级 |
|------|--------|
| 实质性更新（仅改日期不更新内容是违规行为） | **Must-do** |
| 核心页面每 3-6 个月审查更新 | 推荐 |
| 不要为了 SEO 而更新 | 注意 |
| 百度时效性权重 > Google | **Must-do** |

## 5. 内链策略

### Hub-Spoke 模型

```
首页
├── Hub 页（核心专题，集中权重）
│   ├── Spoke A → 链回 Hub + 相关 Spoke
│   ├── Spoke B → 链回 Hub + 相关 Spoke
│   └── Spoke C → 链回 Hub + 相关 Spoke
├── 热点/新闻 → 链接到相关 Hub
└── 博客/知识库 → 链接到产品页
```

### 要点

| 建议 | 优先级 |
|------|--------|
| 锚文本要有意义（告诉搜索引擎目标页是什么） | **Must-do** |
| 重要内容距首页 ≤3 次点击 | 推荐 |
| 正文链接权重 > 侧边栏/底部链接 | 推荐 |
| 每页 3-10 个内部链接为佳 | 推荐 |
| 新内容发布后立即从 2-3 个相关旧文添加内链 | 推荐 |

## 6. 百度特有因素

| 因素 | 优先级 |
|------|--------|
| **ICP 备案**（未备案索引意愿极低） | **Must-do** |
| 百度搜索资源平台验证 | **Must-do** |
| 百度推送 API（加速收录，不直接提排名） | 推荐 |
| **百度统计代码**（信任/活跃度信号，行业公认） | 推荐 |
| HTTPS（百度官方声明是排名信号） | **Must-do** |
| URL 含拼音/中文关键词（比 Google 明显） | 推荐 |
| 中国大陆服务器 | 推荐 |
| 百度智能小程序（URL mapping 替换 H5） | 推荐 |
| 广告合规（冰桶算法打击弹窗） | **Must-do** |

### 百度 vs Google 关键差异

| 维度 | 百度 | Google |
|------|------|--------|
| 备案 | 必须 | 无要求 |
| JS 渲染 | 弱（关键内容必须 SSR） | 强 |
| 时效性权重 | 高 | 中 |
| 结构化数据 | 有限（OpenCard） | 全面（JSON-LD） |
| 外链权重 | 中等 | 高 |
| 用户行为信号 | 影响较大 | 间接 |

## 7. 薄内容惩罚

### 触发条件

| 因素 | 来源 | 优先级 |
|------|------|--------|
| 大量"几乎没有价值"的页面 | Google Spam Policies | **Must-do** |
| Doorway 页面（中间跳转页） | Google Spam Policies | **Must-do** |
| 聚合无增值 | Google Spam Policies | **Must-do** |
| 百度飓风算法（采集内容） | 百度搜索资源平台 | **Must-do** |
| 百度细雨算法（内容极少） | 百度搜索资源平台 | **Must-do** |

### 短页面应对（热点/新闻条目）

| 策略 | 优先级 |
|------|--------|
| 聚合展示为主，短内容不独立 URL（或聚合到列表页） | **Must-do** |
| 添加上下文分析、相关阅读等增值内容 | **Must-do** |
| 每个独立 URL 至少 300-500 字实质性内容 | 推荐 |
| 无法增值的页面用 `noindex` 阻止索引 | 推荐 |
| canonical 指向汇总页 | 推荐 |

## 8. 程序化 SEO

### 适用场景

热点独立页、城市页、标签聚合页、工具模板页

### 最佳实践

| 建议 | 优先级 |
|------|--------|
| 每个程序化页面必须有独特、有用的内容（不只是模板填空） | **Must-do** |
| 避免大量相似页面引导到同一目标（Doorway） | **Must-do** |
| 每页唯一 title + meta description | **Must-do** |
| 至少一个"只有此页面才有"的内容区块 | **Must-do** |
| SSR 渲染（百度兼容） | **Must-do** |
| 程序化页面之间建立语义关联链接 | 推荐 |
| 逐步增加，不要一次性发布大量页面 | 推荐 |

### Google 关于规模化内容的立场

> "Scaled content abuse is when many pages are generated for the primary purpose of manipulating search rankings and not helping users. This abusive practice is typically focused on creating large amounts of unoriginal content that provides little to no value to users, **no matter how it's created**."
>
> — Google Spam Policies

### 程序化页面质量检查

- [ ] 每个页面是否有用户会搜索的独特信息？
- [ ] 页面之间是否有实质性差异（不只是关键词替换）？
- [ ] 内容是否对用户有实际帮助？
- [ ] title 和 meta description 是否唯一且描述性？
- [ ] 是否有内链连接手工编辑的高质量页面？
- [ ] SSR 渲染，百度可抓取？

## 优先级总览

### Must-do

1. 原创内容，不堆砌
2. ICP 备案 + 百度搜索资源平台验证
3. Core Web Vitals 达标
4. 移动端友好
5. HTTPS
6. 每页唯一 title + meta description
7. 作者署名 + 关于页面（E-E-A-T）
8. SSR 关键内容
9. 薄内容聚合处理
10. 程序化页面必须有独特价值

### 推荐

1. 结构化数据（Article, Breadcrumb, FAQ）
2. Hub-Spoke 内链架构
3. 百度统计 + 推送 API
4. 定期内容更新（3-6 月）
5. 图片 alt text
6. 百度智能小程序
7. AI 内容披露

### 不要过度投入

1. 关键词密度精确计算 → 自然写作
2. 追求特定字数 → 无魔法数字
3. 域名堆砌关键词 → 几乎无影响
4. 买卖外链 → 违规
